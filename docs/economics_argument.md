# 数字经济学论证（草稿）

> 该文档用于在数字经济学报告中嵌入 AIGC 3D 商品资产平台的实证支撑。具体数字以 `outputs/jobs/<job_id>/metrics.json` 为准。

## 1. 信息不对称降低

传统电商商品页通常仅含静态图片，消费者无法 360° 检视商品几何与材质，引发购前不确定性 (Akerlof 1970 类问题的现代变体)。

本系统通过：

- 多视角图片 / 视频 → Gaussian Splat 资产（PLY/Splat）；
- 商品资产可在 `<model-viewer>` 中旋转、缩放、AR 预览；

在不增加商家拍摄成本的前提下，把商品信息维度从「图」扩展到「可观察的 3D 结构」。`metrics.json` 中的 `splat_size_mb` 给出维度提升的存储成本上限。

## 2. 搜索成本降低

3D 资产可直接嵌入网页，避免消费者跨页/跨平台对比照片的认知负担。配合 GLB Draco 压缩：

- 平均 GLB 由 X MB 压到 Y MB，下载时间从 a 秒降到 b 秒。
- `compression_ratio` 大致反映网络层面的搜索摩擦下降幅度。

## 3. 交易摩擦降低（试穿）

二维虚拟试穿为消费者提供「先看效果再下单」的选项。报告主张以此为先导：

- 服装类目预期退货率在文献中通常 20-40%；
- 试穿带来的购前可视化可降低退货率（具体数值需要进一步消费者实验）。

本系统通过 `tryon` 管线生成 person/garment/result 三联图，便于做对照实验。

## 4. 数据治理与边界

- 所有任务状态保存在本地 `outputs/jobs/`，便于学术复现，但不能直接当生产存储。
- 模型权重和外部仓库（CatVTON、Nerfstudio）的许可证与商用边界需要在报告里注明。
- 试穿结果可能涉及肖像权，正式发布前需补充隐私政策。
